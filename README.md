## ūüĒć Overview

**ReconScan** is a Python-based domain intelligence and network mapping tool designed for real-world security assessments. It goes far beyond basic subdomain enumeration ‚ÄĒ it probes live HTTP/S endpoints, inspects TLS certificates, scans for open ports, detects WAF/CDN protection, and fingerprints the technology stack of every discovered subdomain.

Built with concurrent execution and real-world accuracy in mind, ReconScan includes a **wildcard DNS false-positive guard** that prevents the flooding of fake results that plague basic tools.

---

## ‚ú® Features

| Feature | Description |
|---|---|
| ūüĆź **WHOIS / Domain Intelligence** | Registrar, creation/expiry dates, name servers, DNSSEC, status codes |
| ūüß© **Subdomain Discovery** | Wordlist-based enumeration with configurable thread count |
| ūüöę **Wildcard DNS Detection** | Probes a random subdomain first to prevent thousands of false positives |
| ūüĆć **IP Geolocation** | Country, city, ASN/ISP via ipinfo.io (no API key required) |
| ūüĒĀ **Reverse DNS** | PTR record lookups for every discovered IP |
| ūüďč **Full DNS Record Extraction** | A, AAAA, CNAME, MX, NS, TXT, SOA per subdomain |
| ūüĆź **HTTP/S Probing** | Live status codes, page titles, redirect chain tracking |
| ūüĒí **TLS Certificate Inspection** | CN, SANs, issuer, expiry, TLS version, cipher suite |
| ūüö™ **Port Scanning** | TCP connect scan on 20 common ports per live host |
| ūüßĪ **WAF / CDN Detection** | Identifies Cloudflare, Akamai, Fastly, AWS CloudFront, Imperva, and more |
| ūüĖ•ÔłŹ **Technology Fingerprinting** | Detects server software, CMS, and frameworks from HTTP headers |
| ūüďä **Multi-format Reporting** | JSON, CSV (Excel-ready), and a dark-themed HTML dashboard |
| ūüďĚ **Audit Logging** | Timestamped `.log` file for every scan session |
| ‚ö° **Concurrent Execution** | ThreadPoolExecutor with configurable thread count |

## ūüõ†ÔłŹ Installation

### Prerequisites

- Python **3.8 or higher**
- pip package manager

### Step 1 ‚ÄĒ Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/reconscan.git
cd reconscan
```

### Step 2 ‚ÄĒ Install dependencies

```bash
pip install -r requirements.txt
```

### Step 3 ‚ÄĒ Add a subdomain wordlist

Place your wordlist at `wordlists/subdomains.txt` (one subdomain prefix per line).

```bash
mkdir wordlists
# Option A: use SecLists (recommended)
curl -o wordlists/subdomains.txt https://raw.githubusercontent.com/danielmiessler/SecLists/master/Discovery/DNS/subdomains-top1million-5000.txt

# Option B: use your own list
echo -e "www\nmail\napi\ndev\nstaging\nadmin" > wordlists/subdomains.txt
```

> If no wordlist is found, the tool falls back to a built-in list of ~50 common subdomains.

## ūüďā Output

All results are saved to the `output/` directory after each scan.

| File | Format | Contents |
|---|---|---|
| `results.json` | JSON | Complete structured data for all subdomains |
| `results.csv` | CSV | Flat table ‚ÄĒ open directly in Excel or Google Sheets |
| `report.html` | HTML | Dark-themed visual dashboard with summary stats |
| `recon_YYYYMMDD_HHMMSS.log` | Log | Full audit trail of the scan session |

### HTML Report Preview

The HTML report includes:
- **Summary cards** ‚ÄĒ total subdomains, live HTTP endpoints, TLS count, WAF/CDN count
- **WHOIS panel** ‚ÄĒ registrar, dates, name servers
- **Full results table** ‚ÄĒ sortable, color-coded, with WAF highlighted in red and wildcard matches in yellow

## ūüĒß Configuration

Edit the constants at the top of `recon_tool.py` to customize behavior:

```python
THREADS      = 25      # Concurrent threads (increase for faster scans, be mindful of rate limits)
TIMEOUT      = 5       # Seconds per network call
MAX_RETRIES  = 2       # Retry attempts on failure
WORDLIST     = "wordlists/subdomains.txt"
OUTPUT_DIR   = "output"

# Ports to scan per discovered subdomain
COMMON_PORTS = [21, 22, 23, 25, 53, 80, 110, 143, 443, 445,
                3306, 3389, 5432, 6379, 8080, 8443, 8888, 9200, 27017]
```
